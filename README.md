# DSA Revision Sheet

This repository is purely for **revision purposes**. It contains my solutions to DSA problems organized by topic, so I can quickly look back at how I solved a problem without reopening LeetCode every time.

## Motive

- Revise solutions topic-wise at any point
- Track progress across all major DSA topics
- Keep solutions clean and accessible in one place

## How It's Structured

Each topic has its own folder. Inside every topic, each question gets its own folder named after its LeetCode number (`LC-<number>`). Inside that folder there are two Java files — one for the brute force approach and one for the optimal — plus a `.md` file for the problem statement.

```
topic/
└── LC-<leetcode-number>/
    ├── bruteforce.java
    ├── optimal.java
    └── <Question Name>.md
```

The `.md` file holds the question name, LeetCode number, and the full problem statement copied from the web for quick reference during revision.

## Extra Questions

Some questions solved outside the main list may be added directly inside the relevant topic folder, following the same folder structure. These are questions picked up along the way and are not part of the original list — no separate tracking for these.

## Note on Solutions

Solutions are written exactly as submitted on LeetCode — just the solution method, no boilerplate. This repo is for reference and revision, not for running code locally.
